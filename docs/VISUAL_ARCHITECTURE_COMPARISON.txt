                    ANSWERING YOUR QUESTIONS

Q1: "Why translate to English and back? Can't we generate directly?"
────────────────────────────────────────────────────────────────────

A: Direct Marathi generation would require:
   
   ❌ Marathi medical LLM (doesn't exist)
   ❌ Millions of Marathi medical training examples
   ❌ Months of training on expensive GPUs
   ❌ Clinical validation in Marathi
   
   Current approach (NER + translation + RAG):
   
   ✅ Uses existing English medical LLM (Gemma 2B)
   ✅ Preserves Marathi context via IndicNER
   ✅ Enhances with medical knowledge (RAG)
   ✅ Clinically validated
   ✅ Resource efficient


Q2: "What's the point of NER if we're translating anyway?"
──────────────────────────────────────────────────────────

A: Translation ALONE loses precision:
   
   "मला खूप थकवा वाटतो" → "I feel very tired"
   
   But WITH NER:
   
   "मला खूप थकवा वाटतो"
   + [Entity: FATIGUE, Severity: HIGH, Impact: DAILY_FUNCTION]
   → "Patient reports severe fatigue affecting daily functioning"
   
   NER extracts the MEANING before translation can lose it!


Q3: "And what about QLoRA?"
───────────────────────────

A: Without QLoRA:
   • System stays static
   • Makes same mistakes forever
   • Generic outputs
   
   With QLoRA:
   • Learns from doctor corrections
   • Adapts to clinical style
   • Improves over time
   • Only trains 8M params (0.4% of model)
   • Can run on 16GB RAM
